# 要約の観点

文書を要約するとき、次の観点で内容を捉える（出力の構造はツールが固定するので、
ここでは「何を・どの観点で拾うか」だけを定める）。該当しない観点は省いてよい。

- 目的・背景: この文書が何のために作られたか
- 結論・要点: 最も重要な結論・決まったこと（最初に述べる）
- 対象範囲: 何を扱い、何を扱わないか
- 仕様・業務ルール: 守るべき仕様・ルール・制約
- 意思決定・合意事項: 決まった事項とその理由
- 課題・リスク・未確定事項: TBD・宿題・懸念
- 関係者・期日: 登場する役割・スケジュール（記載があれば）

推測で補わず、文書に書かれていることだけをまとめる。数値・表は要約でも保持し、
根拠となる章・節・表が分かる場合は該当箇所を添える。
